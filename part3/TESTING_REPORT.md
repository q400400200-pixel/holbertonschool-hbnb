 # Task 6 – API Testing Report

## Overview
This document describes the testing process for the HBnB API endpoints.
All tests were performed using cURL and the Swagger documentation generated by Flask-RESTx.

---

## Tested Endpoints

### 1. Create User
- **Endpoint:** POST /api/v1/users/
- **Input:**
```json
{
  "first_name": "Owner",
  "last_name": "Test",
  "email": "owner.ok@example.com"
}
```
### 2. Create Amenity
- **Endpoint:** POST /api/v1/amenities/
- **Input:**
```json
{
  "name": "WiFi"
}
```
### 3. Create Place

- **Endpoint:** POST /api/v1/places/

- **Input:**
```json
{
  "title": "Nice House",
  "description": "Cozy",
  "price": 120,
  "latitude": 24.7136,
  "longitude": 46.6753,
  "owner_id": "<valid_user_id>",
  "amenities": ["<valid_amenity_id>"]
}
```
### 4. Create Review

- **Endpoint:** POST /api/v1/reviews/

- **Input:**
```json
{
  "text": "Great place!",
  "rating": 5,
  "user_id": "<valid_user_id>",
  "place_id": "<valid_place_id>"
}
```
### 5. Get All Reviews

-**Endpoint:** GET /api/v1/reviews

### 6. Update Review

- **Endpoint:** PUT /api/v1/reviews/<review_id>

- **Input:**
```json
{
  "text": "Updated review",
  "rating": 4
}
```
### 7. Delete Review

-**Endpoint:** DELETE /api/v1/reviews/<review_id
Validation Testing

Invalid email format returns 400 Bad Request

Empty required fields return 400 Bad Request

Rating outside range (1–5) is rejected

Non-existing resources return 404 Not Found

### Issues Encountered

Initial issue with multiple HBnBFacade instances causing "Owner not found"

Issue resolved by using a shared Facade instance across API modules
### Conclusion

All endpoints were successfully tested.
Validation rules worked as expected.
The API meets the functional and validation requirements of Task 6.
